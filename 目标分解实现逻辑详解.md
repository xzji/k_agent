# 目标分解（Goal Decomposition）实现逻辑详解

## 🎯 核心流程

当用户执行 `/goal add <描述>` 时，系统自动进行以下三步操作：

```
用户输入目标
    ↓
1. 创建 GoalNode
    ↓
2. GoalDecomposer.decompose() → 生成维度树
    ↓
3. SourceDiscoverer.discoverAndBind() → 为每个维度绑定信息源
    ↓
完成！目标就绪，后台循环开始工作
```

---

## 📋 1. GoalDecomposer 核心逻辑

### 类结构

```typescript
export class GoalDecomposer {
    private llm: BackgroundLLMChannel;

    // 主方法：将目标分解为维度树
    async decompose(goal: GoalNode): Promise<DimensionNode[]>
}
```

### 分解步骤

#### Step 1: 构建 Prompt

系统会根据用户的目标生成一个详细的中文 prompt：

```
请将以下目标拆解为可以独立探索的维度树。

## 目标
**标题**: 帮我做一份 AI Agent 市场的行业调研报告
**描述**: 帮我做一份 AI Agent 市场的行业调研报告

## 要求
1. 拆解为 5-8 个顶层维度，每个维度可以有 0-3 个子维度
2. 每个维度需要标注：
   - timeliness: "urgent"（需要即时信息）/ "normal" / "can_wait"
   - valueScore: 1-10（对目标完成的价值）
   - maxDepth: 1-3（最大探索深度）

## 输出格式
{
  "dimensions": [
    {
      "title": "市场规模与增长趋势",
      "description": "调研全球 AI Agent 市场规模、增长率、预测",
      "timeliness": "urgent",
      "valueScore": 9,
      "maxDepth": 2,
      "children": [...]
    }
  ]
}
```

**关键设计点**：
- ✅ 使用 **中文 prompt**，提高 LLM 理解准确性
- ✅ 明确输出 **5-8 个维度**（避免过多或过少）
- ✅ 要求标注 **时效性**、**价值分**、**探索深度**
- ✅ 支持**层级结构**（子维度）

#### Step 2: 调用 LLM

```typescript
const result = await this.llm.chatJSON<{ dimensions: DecomposedDimension[] }>({
    systemPrompt: "You are a strategic planning assistant...",
    messages: [{ role: "user", content: prompt }],
    temperature: 0.5,  // 降低随机性，提高稳定性
});
```

**技术细节**：
- 使用 `chatJSON<T>` 方法，要求 LLM 返回纯 JSON
- `temperature: 0.5`：在创造性和稳定性之间平衡
- 如果解析失败，有 **fallback 机制**（创建通用维度）

#### Step 3: 转换为 DimensionNode 树

```typescript
private convertToNodes(goalId, dims, parentId): DimensionNode[] {
    return dims.map(dim => {
        const node = {
            id: generateId(),
            goalId,
            parentDimensionId: parentId,
            title: dim.title,
            description: dim.description,
            status: "pending",           // 初始状态
            explorationDepth: 0,         // 探索深度从0开始
            maxDepth: dim.maxDepth,
            timeliness: dim.timeliness,
            valueScore: dim.valueScore,
            children: [],
            dataSources: [],             // 稍后由 SourceDiscoverer 填充
            createdAt: Date.now(),
            updatedAt: Date.now(),
        };

        // 递归处理子维度
        if (dim.children?.length > 0) {
            node.children = this.convertToNodes(goalId, dim.children, node.id);
        }

        return node;
    });
}
```

## 🌲 2. 维度树结构

### 实际例子

假设用户创建目标："调研 AI Agent 框架"，系统可能会生成：

```
目标: 调研 AI Agent 框架
│
├─ [维度1] 市场规模与增长 (v=9, urgent, d=2)
│  ├─ [子维度1.1] 全球市场规模
│  └─ [子维度1.2] 年增长率预测
│
├─ [维度2] 主要玩家分析 (v=8, normal, d=3)
│  ├─ [子维度2.1] 开源框架
│  ├─ [子维度2.2] 商业化产品
│  └─ [子维度2.3] 技术对比
│
├─ [维度3] 技术路线对比 (v=7, normal, d=2)
│
├─ [维度4] 应用场景分析 (v=6, can_wait, d=2)
│
└─ [维度5] 发展趋势预测 (v=7, normal, d=2)
```

### 维度属性说明

| 属性 | 类型 | 说明 | 举例 |
|------|------|------|------|
| `title` | string | 维度标题 | "市场规模与增长" |
| `description` | string | 详细描述 | "调研全球市场规模..." |
| `timeliness` | urgent/normal/can_wait | 时效性 | urgent=需要即时信息 |
| `valueScore` | 1-10 | 价值分 | 9=对目标很重要 |
| `maxDepth` | 1-3 | 最大探索深度 | 3=需要深入研究 |
| `status` | pending/exploring/explored | 探索状态 | 初始为 pending |
| `explorationDepth` | 0-maxDepth | 当前探索深度 | 每执行一次+1 |

## 🔗 3. 与 SourceDiscoverer 的集成

### 自动信息源发现

维度树生成后，系统会自动为每个维度发现信息源：

```typescript
// GoalManager.createGoal() 中的流程
const decomposer = new GoalDecomposer(this.llm);
const initialDimensions = await decomposer.decompose(goal);  // ← 拆解
this.dimensions.set(goal.id, initialDimensions);

// 自动为每个维度绑定信息源
const sourceDiscoverer = new SourceDiscoverer(this.llm);
const sources = await sourceDiscoverer.discoverAndBind(goal, initialDimensions);
```

### 信息源发现逻辑

`SourceDiscoverer` 会：
1. 分析每个维度的主题
2. 推荐专业的信息源（arXiv、GitHub、行业网站等）
3. 检查可达性（HEAD 请求、API probe）
4. 绑定到维度（`dimension.dataSources`）

## 🧠 4. LLM Prompt 设计哲学

### 为什么用中文 Prompt？

1. **准确性**：LLM 对中文语境理解更准确
2. **领域知识**：能更好识别中文技术术语和概念
3. **格式控制**：JSON 结构要求更清晰

### Prompt 的关键要素

```typescript
// 目标信息
标题: ${goal.title}
描述: ${goal.description}
约束: ${goal.constraints}

// 输出要求
1. 数量控制：5-8个维度
2. 属性标注：timeliness, valueScore, maxDepth
3. 层级支持：children 数组

// 输出格式
严格的 JSON Schema
```

## 🔄 5. 动态维度扩展

维度树不是一成不变的！系统支持动态扩展：

### 何时添加新维度？

1. **推理过程中发现**：ActionReasoner 推理时发现新角度
2. **执行过程中发现**：Executor 执行时发现新方向
3. **相关性裁决时发现**：RelevanceJudge 判断时发现新线索

### 添加方法

```typescript
// 在后台循环中
await this.goalManager.addDimensions(goal.id, [
    {
        title: "新发现的维度",
        description: "...",
        timeliness: "urgent",
        valueScore: 7,
        maxDepth: 2,
        discoveredDuring: "action_reasoning"  // 标记发现来源
    }
]);
```

## ⚙️ 7. 核心代码路径

```
用户: /goal add <描述>
    ↓
extension.ts: add handler
    ↓
GoalManager.createGoal()
    ├─ 创建 GoalNode
    ├─ GoalDecomposer.decompose()  ← 这里！
    │   ├─ buildDecomposePrompt()     构建 prompt
    │   ├─ llm.chatJSON()            调用 LLM
    │   ├─ convertToNodes()           转换为树结构
    │   └─ return DimensionNode[]
    ├─ SourceDiscoverer.discoverAndBind()
    └─ persistGoal()                  持久化到磁盘
    ↓
BackgroundAgentLoop.start()
    ↓
后台循环开始按维度探索...
```

## 🎓 8. 设计亮点

1. **全自动**：用户只需一句话，系统自动分解、发现信息源
2. **层次化**：维度树支持多层级，可以深入探索
3. **动态扩展**：运行中发现新维度可以随时添加
4. **容错性**：LLM 解析失败时有 fallback 机制
5. **持久化**：所有维度自动保存到 `~/.pi/agent/goal-driven/goals/`

## 💡 10. 可扩展性

### 如何调整分解策略？

**修改 prompt 模板**：
```typescript
// 在 goal-decomposer.ts 中
private buildDecomposePrompt(goal: GoalNode): string {
    // 可以根据 goal.type 使用不同的 prompt 模板
    // 可以调整维度数量、深度要求等
}
```

### 如何支持新的维度属性？

```typescript
// 1. 在 types.ts 中添加属性
interface DimensionNode {
    // ... 现有属性
    customField?: string;  // 新字段
}

// 2. 在 prompt 中要求 LLM 生成
"customField": "用户自定义字段"

// 3. 在 convertToNodes 中映射
customField: dim.customField ?? ""
```

## 总结

**目标分解的核心** = LLM 理解 + 结构化输出 + 树形组织 + 自动信息源绑定

通过精心设计的中文 prompt，系统将用户的一句话目标自动拆解为可执行、可追踪、可扩展的维度树，为后续的自主探索奠定基础。
